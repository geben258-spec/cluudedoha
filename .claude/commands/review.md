# 품질 검사 (QC)

작업 완료를 보고하기 전, 반드시 이 검사를 수행하세요.

## Step 1: 변경 사항 확인
`git diff`를 실행하여 수정된 파일 목록을 확인합니다.

## Step 2: 셀프 체크 (5가지 항목)
각 항목을 점검하고 점수를 매기세요:

1. **정확성 (30점)**: 요구사항을 정확히 구현했는가?
2. **가독성 (20점)**: 코드가 읽기 쉽고 명명이 적절한가?
3. **성능 (15점)**: 불필요한 연산, N+1 쿼리 등이 없는가?
4. **보안 (20점)**: XSS, SQL Injection, 민감정보 노출이 없는가?
5. **매뉴얼 준수 (15점)**: manuals/ 규칙을 따랐는가?

## Step 3: 보고서 작성
`docs/REVIEW.md`에 다음 형식으로 저장:

```markdown
# 코드 리뷰 결과
**날짜**: YYYY-MM-DD
**총점**: XX/100

## 변경 파일
- (파일 목록)

## 잘한 점
- (구체적 칭찬)

## 수정 필요
- [심각] 파일:라인 - 설명 → 수정 방법
- [권장] 파일:라인 - 설명 → 수정 방법

## 판정
- 🟢 통과 (90+) / 🟡 조건부 (70-89) / 🔴 반려 (70 미만)
```

## Step 4: 미흡한 점 수정
90점 미만이면 즉시 수정하고 재검사.
보안 이슈 발견 시 점수와 무관하게 즉시 수정.
