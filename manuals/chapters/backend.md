# Chapter 3: 백엔드 가이드라인

**키워드**: API, 서버, 라우트, 인증, 엔드포인트, 미들웨어

## 3.1 API 설계
- RESTful 원칙 준수
- 엔드포인트 명명: 복수형 명사 사용 (`/users`, `/posts`)
- HTTP 상태 코드 정확히 사용 (200, 201, 400, 401, 403, 404, 500)
- 응답 형식 통일: `{ "data": ..., "error": null }`

## 3.2 에러 처리
- 모든 에러는 중앙 에러 핸들러에서 처리
- 사용자에게는 안전한 에러 메시지만 노출
- 내부 에러는 로그에 상세히 기록

## 3.3 인증/인가
- JWT 기반 인증 사용
- 토큰 만료 시간 설정 필수
- 민감한 엔드포인트에는 권한 검사 미들웨어 적용

## 3.4 성능
- N+1 쿼리 방지
- 페이지네이션 필수 (기본 limit: 20)
- 캐싱 전략 명시 (메모리/Redis)
