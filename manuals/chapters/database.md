# Chapter 4: 데이터베이스 가이드라인

**키워드**: DB, 모델, 스키마, 마이그레이션, 쿼리, 테이블

## 4.1 스키마 설계
- 테이블명: 복수형 snake_case (`users`, `chat_logs`)
- 컬럼명: snake_case (`created_at`, `user_id`)
- 모든 테이블에 `id`, `created_at`, `updated_at` 포함

## 4.2 마이그레이션
- 스키마 변경은 반드시 마이그레이션 파일로 관리
- 마이그레이션은 롤백 가능해야 함
- 프로덕션 DB에 직접 쿼리 실행 금지

## 4.3 쿼리 규칙
- ORM 사용 우선, Raw SQL은 성능 최적화 시에만
- SELECT *  사용 금지 (필요한 컬럼만 명시)
- 인덱스 설계 필수 (자주 검색/정렬하는 컬럼)

## 4.4 데이터 무결성
- 외래키 제약조건 설정
- NOT NULL 적극 활용
- 소프트 삭제(soft delete) 우선 고려
